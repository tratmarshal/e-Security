# Warrant Tracker

บริบทนี้อธิบายภาษากลางของระบบติดตามหมายจับสำหรับเจ้าหน้าที่ตำรวจศาล ตั้งใจให้ใช้แยกความหมายของหมายจับ เหตุที่ได้ตัวจำเลย และงานธุรการหลังจากหมายจับสิ้นผล

## Language

**เจ้าหน้าที่ตำรวจศาล**:
ผู้ใช้งานที่ได้รับอนุญาตให้ค้นหาและจัดการข้อมูลหมายจับในระบบนี้ เจ้าหน้าที่ตำรวจศาลทุกคนในระบบมีสิทธิ์เท่ากัน
_Avoid_: แอดมิน, ผู้ดูแลระบบ

**จำเลย**:
บุคคลที่ถูกระบุอยู่ในหมายจับและต้องกลับเข้าสู่กระบวนการพิจารณาคดี จำเลยหนึ่งคนอาจมีหมายจับได้หลายรายการ
_Avoid_: ผู้ต้องหา, ผู้ถูกจับ

**หมายจับ**:
รายการตามคำสั่งศาลที่ระบุว่าจำเลยยังต้องเข้าสู่กระบวนการตามคดี จำเลยหนึ่งคนอาจมีหลายหมายจับ และแต่ละหมายจับมีสถานะของตัวเอง
_Avoid_: คดี, สำนวน, รายการดำเนินการ

**เลขที่หมายจับ**:
ตัวระบุหมายจับหนึ่งรายการ เลขที่หมายจับต้องอ้างถึงหมายจับได้เพียงรายการเดียวในระบบ
_Avoid_: เลขคดี, เลขสำนวน, รหัสรายการ

**ฐานข้อมูลหมายจับ**:
Spreadsheet หลักที่เก็บข้อมูลหมายจับตั้งแต่ปี 55 ถึงปัจจุบัน เป็นแหล่งอ้างอิงของรายการหมายจับและข้อมูลจำเลยในบริบทนี้
_Avoid_: รายการอัปเดต, ฐานข้อมูลรวม

**รายการอัปเดต**:
Spreadsheet แยกจากฐานข้อมูลหมายจับ ใช้เก็บรายการแก้ไขที่รับมาจากหน้าระบบและประวัติการใช้งานของเจ้าหน้าที่
_Avoid_: ฐานข้อมูลหมายจับ, ประวัติการใช้งาน

**รายการแก้ไข**:
ข้อมูลการแก้ไขหมายจับที่หน้าระบบส่งเข้ามาเพื่อให้ฐานข้อมูลหมายจับอัปเดตตามรายการนั้น หน้าระบบอ่านรายการแก้ไขกลับไปแสดงผลได้ แต่รายการแก้ไขเป็นพื้นที่พักข้อมูล ไม่ใช่ฐานข้อมูลหมายจับโดยตรง
_Avoid_: หมายจับ, ประวัติการใช้งาน

**รายการแก้ไขล่าสุด**:
รายการแก้ไขที่ถือเป็นผลล่าสุดของหมายจับหนึ่งรายการ หากมีหลายรายการแก้ไขอ้างถึงหมายจับเดียวกัน รายการแก้ไขล่าสุดเป็นรายการที่ใช้สะท้อนลงฐานข้อมูลหมายจับ
_Avoid_: รายการแก้ไขทั้งหมด, ประวัติการใช้งาน

**ลำดับรายการแก้ไข**:
ตัวเรียงลำดับที่ระบบสร้างให้รายการแก้ไข ใช้ตัดสินว่ารายการแก้ไขใดเป็นรายการล่าสุดเมื่อมีหลายรายการอ้างถึงหมายจับเดียวกัน
_Avoid_: เวลาบันทึกเป็นตัวตัดสินหลัก, ลำดับในหน้าจอ

**จุดสถานะรายการแก้ไข**:
สัญลักษณ์บนหน้าระบบที่บอกว่ารายการแก้ไขสะท้อนลงฐานข้อมูลหมายจับแล้วหรือยัง จุดสีเขียวหมายถึงสะท้อนแล้ว และจุดสีแดงหมายถึงยังไม่สะท้อนหรือสะท้อนไม่สำเร็จ หน้าระบบแสดงเป็นจุดสีเท่านั้น แม้พื้นที่รายการอัปเดตจะมีค่าสถานะภายในสำหรับระบบ
_Avoid_: สถานะหมายจับ, สถานะสำนวน

**สถานะภายในของรายการแก้ไข**:
ค่าสถานะที่ระบบใช้ติดตามว่ารายการแก้ไขสะท้อนลงฐานข้อมูลหมายจับแล้วหรือยัง มีค่า `pending`, `synced`, และ `error` โดยผู้ใช้ทั่วไปเห็นผลเป็นจุดสถานะรายการแก้ไขแทนข้อความเหล่านี้
_Avoid_: สถานะหมายจับ, สถานะสำนวน

**งานอัปเดตอัตโนมัติ**:
งานเบื้องหลังที่ปรับฐานข้อมูลหมายจับตามรายการแก้ไขล่าสุดแบบเรียลไทม์ โดยไม่ใช่งานที่ผู้ใช้เรียกจากหน้าระบบโดยตรง
_Avoid_: คำสั่งหน้าระบบ, การค้นหา

**สถานะหมายจับ**:
สถานะหลักของหมายจับแต่ละรายการ ไม่ใช่สถานะของจำเลย มีลำดับการเปลี่ยนสถานะทางเดียวคือ `ต้องการตัว` → `สิ้นผลรอเพิกถอน` → `เพิกถอน`
_Avoid_: สถานะสำนวน, เหตุ

**ต้องการตัว**:
สถานะหมายจับที่หมายความว่ายังไม่ได้ตัวจำเลยหรือยังไม่พบเหตุที่ทำให้หมายจับสิ้นผล เป็นสถานะเดียวที่อนุญาตให้บันทึกเหตุได้ตัว
_Avoid_: เปิดใช้งาน, ยังไม่จบ

**สิ้นผลรอเพิกถอน**:
สถานะหมายจับที่หมายความว่าหมายจับสิ้นผลแล้วเพราะมีเหตุที่ทำให้จำเลยกลับเข้าสู่กระบวนการพิจารณาคดี แต่ยังไม่ได้บันทึกการเพิกถอน
_Avoid_: ได้ตัว, รอปิดงาน, รอเพิกถอน

**รอเพิกถอน**:
ป้ายแสดงผลในรายการแก้ไขที่หมายถึงสถานะเดียวกับ `สิ้นผลรอเพิกถอน` ในฐานข้อมูลหมายจับ ใช้ในพื้นที่รายการอัปเดตเท่านั้น ไม่ใช่ชื่อสถานะหมายจับหลัก
_Avoid_: สิ้นผลรอเพิกถอนในฐานข้อมูลหมายจับ, เพิกถอน

**ค่าหลักของฐานข้อมูลหมายจับ**:
ค่าที่ฐานข้อมูลหมายจับใช้บันทึกจริง แม้รายการแก้ไขหรือหน้าระบบจะแสดงคำที่สั้นกว่า งานอัปเดตอัตโนมัติต้องแปลงค่าจากพื้นที่ทำงานให้เป็นค่าหลักก่อนเขียนฐานข้อมูลหมายจับ
_Avoid_: ป้ายแสดงผล, ค่าหน้าระบบ

**เพิกถอน**:
สถานะหมายจับที่หมายความว่าหมายจับถูกบันทึกว่าเพิกถอนแล้ว สถานะนี้เป็นปลายทางของหมายจับในบริบทนี้
_Avoid_: ปิดงาน, ลบหมายจับ

**เหตุ**:
เหตุผลที่ทำให้จำเลยกลับเข้าสู่กระบวนการพิจารณาคดีและทำให้หมายจับสิ้นผล เหตุอธิบายว่าสถานะหมายจับเปลี่ยนเป็น `สิ้นผลรอเพิกถอน` เพราะอะไร เช่น `มอบตัว`, `ตำรวจจับ`, `อยู่เรือนจำอื่น`, หรือ `อื่น ๆ`
_Avoid_: สถานะหมายจับ, สถานะสำนวน

**อยู่เรือนจำอื่น**:
เหตุที่หมายความว่าจำเลยถูกควบคุมตัวอยู่ในเรือนจำจากคดีหรือหมายอื่นแล้ว จึงถือว่าได้ตัวในทางกระบวนการ แม้ไม่ได้จับตัวจากหมายจับรายการนี้โดยตรง
_Avoid_: ตำรวจจับ, เพิกถอน

**ตำรวจจับ**:
เหตุที่หมายความว่าตำรวจจับจำเลยได้ตามหมายจับรายการนี้โดยตรง
_Avoid_: อยู่เรือนจำอื่น, มอบตัว

**มอบตัว**:
เหตุที่หมายความว่าจำเลยมาพบเจ้าหน้าที่หรือศาลเองโดยสมัครใจ ไม่ใช่การถูกจับตามหมายจับและไม่ใช่การถูกควบคุมตัวจากคดีหรือหมายอื่น
_Avoid_: ตำรวจจับ, อยู่เรือนจำอื่น

**อื่น ๆ**:
เหตุสำรองสำหรับกรณีที่ `มอบตัว`, `ตำรวจจับ`, และ `อยู่เรือนจำอื่น` ไม่ครอบคลุม ต้องมีรายละเอียดเหตุเสมอ
_Avoid_: หมายเหตุทั่วไป

**รายละเอียดเหตุ**:
คำอธิบายเพิ่มเติมของเหตุ ใช้เมื่อเหตุหลักคือ `อื่น ๆ` หรือเมื่อเจ้าหน้าที่ต้องการบอกบริบทที่เหตุหลักไม่ครอบคลุม
_Avoid_: หมายเหตุทั่วไป

**สถานะสำนวน**:
สถานะงานธุรการของรายการดำเนินการ มีลำดับการเปลี่ยนสถานะคือ `รอดำเนินการ` → `เสนอศาล` → `รอส่งต่อ` → `ส่งต่อแล้ว` สถานะสำนวนไม่ใช่สถานะหมายจับ
_Avoid_: สถานะหมายจับ, เหตุ, ดำเนินการเสร็จแล้ว

**รอดำเนินการ**:
สถานะสำนวนเริ่มต้นของหมายจับที่ศาลออกแล้วและหมายจับยังอยู่ในสถานะ `ต้องการตัว`
_Avoid_: ต้องการตัว, รายการที่ดำเนินอยู่

**เสนอศาล**:
สถานะสำนวนที่หมายความว่าพบเหตุที่ทำให้หมายจับสิ้นผลแล้วและอยู่ระหว่างเสนอศาลเพื่อพิจารณาขั้นต่อไป
_Avoid_: สิ้นผลรอเพิกถอน, รอส่งต่อ

**รอส่งต่อ**:
สถานะสำนวนที่หมายความว่าสำนวนกลับมาที่ตำรวจศาลแล้วและรอดำเนินการส่งต่อหลังจากศาลพิจารณา
_Avoid_: เพิกถอน, ส่งต่อแล้ว

**ส่งต่อแล้ว**:
สถานะสำนวนที่หมายความว่างานในบริบทหมายจับส่งต่อไปยังกระบวนการถัดไปแล้ว ไม่ได้หมายความว่ากระบวนการทั้งหมดของคดีเสร็จสิ้น
_Avoid_: ดำเนินการเสร็จแล้ว, ปิดคดี

**รายการดำเนินการ**:
งานที่เกิดขึ้นหลังจากได้ตัวจำเลยหรือพบเหตุที่ทำให้หมายจับสิ้นผล หนึ่งรายการดำเนินการอ้างถึงหมายจับหนึ่งรายการ แม้เจ้าหน้าที่จะเลือกหลายหมายจับพร้อมกันด้วยเหตุเดียวกัน แต่ละหมายจับยังเป็นรายการดำเนินการแยกกัน
_Avoid_: หมายจับ, ประวัติการใช้งาน

**รายการที่ดำเนินอยู่**:
รายการดำเนินการที่ยังต้องทำงานธุรการต่อจนกว่าสำนวนจะถูกส่งต่อแล้ว รายการที่ดำเนินอยู่รวมถึงรายการที่หมายจับสิ้นผลแล้ว รอเพิกถอน หรือเพิกถอนแล้วแต่ยังไม่ได้ส่งต่อสำนวน
_Avoid_: ประวัติรายการดำเนินการ, รายการเพิกถอนแล้ว

**ประวัติรายการดำเนินการ**:
รายการดำเนินการที่สำนวนถูกส่งต่อแล้วและหมายจับอยู่ในสถานะ `เพิกถอน` แสดงผลเพื่อการตรวจสอบย้อนหลังเท่านั้น และไม่สามารถทำรายการต่อได้
_Avoid_: รายการที่ดำเนินอยู่

**การเลือกหมายจับ**:
การระบุว่าหมายจับรายการใดบ้างสิ้นผลจากเหตุเดียวกัน เจ้าหน้าที่อาจเลือกทีละหมายจับหรือเลือกหลายหมายจับพร้อมกันได้ หากเหตุเดียวกันทำให้หมายจับเหล่านั้นสิ้นผลจริง
_Avoid_: เลือกจำเลย, เหมารวมทุกหมายจับ

**ประวัติการใช้งาน**:
บันทึกว่าเจ้าหน้าที่ตำรวจศาลทำรายการใด เมื่อใด และเข้าจากหน้าใดของระบบ เพื่อใช้ตามร่องรอยการใช้งานย้อนหลัง
_Avoid_: รายการดำเนินการ, หมายเหตุ
